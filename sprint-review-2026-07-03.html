Katch Cloud — Milestone 20 Review

Milestone 20 · June 22 – July 3, 2026 · End-of-sprint snapshot · Generated July 6, 2026

Sprint Headline — Exclusive-contracts testing drove Venue Collections to close; five of six themes delivered, Booths API slips to M22

Milestone 20 closed at 92% (91 of 98 milestone items) with 7 items carrying to M22. On the numbers that matter for throughput this is the strongest sprint in the run — 91 items closed against the biggest scope we've carried (98, up from 68 the prior fortnight), even as the headline percentage eased from the previous sprint's 95%. The fortnight was built around a six-theme brief, and five landed. Testing the exclusive-contracts feature did exactly what it should — it pushed the Venue Collections epic (#5173) to close and surfaced a wave of dev-test-feedback fixes. DEV-OC + NetSuite-mock correctness (#5332, #5353), the RainForest→Playwright migration (#5204, #5232, #5233), a trunk + LaunchDarkly dark-launch release process (#5117), and weekly AWS cost-reduction in the infra repo (cost-opt #481–#484) all shipped. The one miss: Booths API discovery produced no tracked work — it moves to M22 as the headline.

Watch item — the exclusive-contracts feature isn't signed off yet, and scope grew to 98

Two things to put a minute on at retro. First, "test exclusive contracts" was the #1 goal and testing worked — but it surfaced a dev-test-feedback tail that is carrying: the polish epic (#5346) plus real bugs including a wrong NetSuite invoice date on venue-collection completion (#5382), a rental-only validation rejecting venue collections (#5365), and a SKU backfill (#5318). The core is in and exercised, but "tested / done" needs a definition and a sign-off owner before M22 builds on it. Second, the milestone grew from ~70 to 98 items across the fortnight — healthy in that testing found real work, but a scope-freeze data point worth checking: did the Day-8 freeze hold, or did fixes pull in uncontrolled?

What We Shipped — Themes

Grouped by theme, not by individual. The fortnight ran against a six-part brief. Five delivered — exclusive-contracts testing and Venue Collections, DEV-OC / NetSuite-mock correctness, the RainForest→Playwright migration, the LaunchDarkly dark-launch release process, and weekly AWS cost-reduction — plus the usual refills, access-control and dependency tail. Booths API discovery did not start and moves to M22.

Theme 1 — Exclusive contracts & Venue Collections: tested to close

The milestone's #1 brief — "test the exclusive-contracts feature" — drove the Venue Collections epic (#5173) to close (feature PR #5294), including the Venue Collection Fee column on market-SKU pricing for VENUE_PICKUP SKUs (#5245) and the Extend Contract pricing-engine flag in RentalPricingService (#4905). Exercising the feature end-to-end surfaced a batch of dev-test feedback, most of which was fixed in-sprint; the remainder is tracked under polish epic #5346 and carries to M22.

Theme 2 — DEV-OC & NetSuite: the mock/sync now behaves like a real environment

The "improve DEV-OC testing + NS mock" brief produced the reliability fixes that make dev-oc trustworthy. EventBus and NetSuite were treating dev-oc as production because they keyed off APP_ENV instead of the deployment tier (#5332, fix PR #5337) — dev-oc was silently dropping domain events and blocking NetSuite capture. Separately, completing an order 500'd the operator when the dev-oc NS sync had no token; post-completion sync now fails soft (#5353, test PR #5367). NetSuite was the single biggest label this sprint (27 closed).

Theme 3 — Test & release enablers: Playwright migration + LaunchDarkly dark-launch

Two enabler themes landed together. The RainForest→Playwright migration moved from pilot to real coverage — the Playwright e2e suite + Rainforest pilot port (#5204), then Phase 2 (active regression / access-control, lifecycle & memberships specs, #5232 / PR #5236) and Phase 3 (Rentals suite, #5233). Separately, the team adopted a trunk + LaunchDarkly dark-launch release process (#5117 / PR #5119) — region-scoped dev deploys, docs, and a release skill — the groundwork for the faster release cadence M22 wants.

Theme 4 — AWS cost-reduction (infra repo) + refills / access polish

The "weekly AWS changes for cost reduction" brief ran as a cost-opt series in the kubernetes repo: right-size dev pod requests (#481 / PR #482), shrink dev node groups (spot 3→1, stateful xlarge→large, #483), and dev off-hours node scale-down (#484 — a decision still to confirm), plus per-environment APP_LOCATION in the Helm chart (#477). On the app side, a steady refills / access-control tail closed: SUPPORT operators can manage org memberships (#5240 / PR #5241), refill period vs scheduled date (#5237), and a refill-permissions inconsistency (#5184).

Also notable — dependency & tooling hygiene kept pace

Alongside the feature work, the team kept the maintenance surface clean: a guzzlehttp/guzzle bump 7.10.0→7.12.1 (#5216) and a new konvoy-dependabot-review skill for automated backwards-compat review of Dependabot PRs. With refills now generating live NetSuite invoices and a Symfony 8 / PHP 8.5 upgrade looming in M22, keeping dependencies current and automatically reviewed is exactly the right muscle to be building now.

Retrospective — Team Themes

Three columns: what worked, what to review, and open questions for the M22 kickoff. Team-level only.

What worked

  • Highest output of the run. 91 closed on 98 scope — the team scaled throughput without dropping the percentage far.
  • Five of six themes delivered. Contracts testing, DEV-OC/NS mock, Playwright, LaunchDarkly and AWS cost-opt all landed.

What to review

  • Booths API didn't start. A stated goal with zero tracked work — was it deprioritised deliberately, or squeezed out?

  • Scope grew to 98. Testing pulled in fixes — did the Day-8 freeze hold?
  • Issue/PR hygiene. #5227's merged-but-open state shouldn't recur — confirm auto-close links.

Open questions for M22

  • Booths API is now the M22 headline — does it get a timeboxed discovery spike before build commitment?
  • Symfony 8 / PHP 8.5 prep — what's the blast radius, and is Playwright coverage ready to catch regressions?
  • Playwright into CI + dev-oc — reserve capacity or it slips, as enablers do.
  • Who signs off the exclusive-contracts feature as tested?
